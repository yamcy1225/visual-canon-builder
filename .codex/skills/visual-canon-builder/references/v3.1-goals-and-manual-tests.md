# v3.1 Goals And Manual Tests

Use this reference when testing whether `$visual-canon-builder` preserves canon certainty, proportion safety, and `$imagegen` handoff discipline on realistic inputs.

## Registered Goals

1. Separate `observed` visual facts from `confirmed` canon.
2. Add an explicit `ready | provisional | blocked` handoff decision table.
3. Use per-landmark or per-envelope projection widths instead of one global character width.
4. Record yaw direction and subject-left/subject-right visibility for asymmetric details.
5. Replace generic `Constraints` with `Confirmed constraints`, `Provisional constraints`, and `Avoid`.
6. Use numeric tolerance only after calibrated reference-sheet measurements.
7. Give faction/world ontologies the same semantic mapping, provenance, and validation-shape structure as character ontologies.
8. Maintain manual golden tests with realistic image-derived cases.

### Pass Criteria

- `Confirmed constraints` remains empty or minimal until the source is explicitly approved.
- `Provisional constraints` contains observed but unapproved image-derived details.
- `Handoff status` is `provisional`, not `ready`, while canon approval and permanent-prop status remain unresolved.
- `Handoff status` is not `blocked` when exactly one usable canon candidate exists and no identity-critical conflict exists.
- Projection uses `derived_projected_widths` per landmark/envelope, not a single `derived_projected_width`.
- The skateboard action pose is marked scene-specific and low-confidence for measurement.
- Numeric tolerance is not applied before calibration: `calibration_status: uncalibrated` and `tolerance_applies: false`.
- Yaw direction and visible subject side are explicit in `view_spec`.
- `Text (verbatim)` is empty or explicitly provisional until shirt text is confirmed or requested as exact output text.
- The prompt pack uses `Confirmed constraints`, `Provisional constraints`, `Unresolved questions`, and `Avoid`; it does not use a generic `Constraints` field.

## Manual Test: Faction Provenance Coverage

Use this text-only fixture to verify Goal 7 without needing images.

### Input

```text
Create a faction visual canon for "Moon Archive".
Image A is the approved canon emblem sheet.
Image B is a winter festival variant.
Image C is only a style reference for brush texture.
The faction's primary color is deep indigo and its required symbol is a crescent archive seal.
```

### Expected Excerpt

```yaml
entity_type: Faction
id: FCT_MoonArchive

semantic_mapping:
  class: Faction
  individual: FCT_MoonArchive
  property_model: visual_language
  relation_model: relations
  constraint_model: validation_shapes
  provenance_model: canon_assertions

relations:
  - subject: Image_A
    predicate: depictsFactionVisualLanguage
    object: FCT_MoonArchive
  - subject: Image_B
    predicate: variantOf
    object: FCT_MoonArchive
  - subject: Image_C
    predicate: styleReferenceFor
    object: FCT_MoonArchive

canon_assertions:
  - id: ASSERT_FCT_PrimaryColor
    subject: FCT_MoonArchive
    predicate: hasPalette
    object: deep_indigo
    source_image_id: Image_A
    source_role: approved canon source
    confidence: observed
    canon_status: confirmed
    asserted_by: visual-canon-builder
    derived_from: faction_fixture
    needs_confirmation: false
  - id: ASSERT_FCT_BrushTexture
    subject: FCT_MoonArchive
    predicate: hasBrushTexture
    object: loose_ink_brush_texture
    source_image_id: Image_C
    source_role: style reference
    confidence: observed
    canon_status: provisional
    asserted_by: visual-canon-builder
    derived_from: faction_fixture
    needs_confirmation: true

validation_shapes:
  - target: FCT_MoonArchive
    path: visual_language.palette.primary
    constraint: equals deep_indigo
    severity: reject
    message: 세력 대표 색상이 deep_indigo가 아니면 정본 위반이다.
```

### Pass Criteria

- Faction output includes `semantic_mapping`, `relations`, `canon_assertions`, and `validation_shapes`.
- `source_role` accepts `approved canon source`, `variant`, and `style reference` without lossy relabeling.
- Style-reference-only facts remain `canon_status: provisional`.
- Faction validation shapes use the shared severity vocabulary: `pass | fix | reject | needs_confirmation | blocked`.
