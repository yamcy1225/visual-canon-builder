# v3.6 Style Fidelity Tests

Use these tests when `$visual-canon-builder` creates `$imagegen` prompt packs from flat mascot, cartoon, sprite, icon, or reference-sheet art. The goal is to prevent correct identity with wrong rendering style.

## Goals

1. Add `style_fidelity_model` when a reference image has a distinctive art style.
2. Add `Style fidelity lock` to every `$imagegen` prompt pack when style must match the reference.
3. Treat semi-realistic rendering, 3D render shading, realistic product details, or adult/fashion proportions as drift unless explicitly requested.
4. Keep style drift out of `Confirmed constraints`; put it in `Avoid` and validation checks.
5. Reject generated outputs that preserve text/identity but lose the reference-sheet style or mascot proportions.

## SampleMascot Regression

Input: approved SampleMascot canon sheet, then a request for an angry crossed-arm image with shirt text `MAD`.

Observed bad output pattern:

- Correct animal identity and `MAD` text.
- Body becomes too tall and mature compared with the reference sheet.
- Legs and torso stretch toward fashion/athletic illustration proportions.
- Shoes become overly realistic product-render sneakers.
- Shading becomes smooth semi-realistic/3D-like grayscale rather than flat sheet line art.

Expected prompt pack hardening:

```text
Style fidelity lock: strict match to the approved reference sheet; flat 2D black-and-white mascot line art; clean bold outline; simple interior lines; minimal flat grayscale only; no semi-realistic shading, no 3D render lighting, no realistic sneaker/product detail, no fashion-illustration anatomy.
View/proportion lock: compact cute mascot proportions from the sheet; large head, compact torso, short simplified limbs, simplified shoes; no elongated adult legs or athletic-fashion body.
Avoid: semi-realistic character art, 3D render, airbrushed shadows, realistic fur texture, realistic sneaker detail, elongated adult proportions, fashion illustration pose anatomy.
```

Expected validation:

- If the generated image has correct `MAD` text but semi-realistic proportions/rendering, verdict is `reject`, not `fix`.
- If only the pose is slightly off while style/proportion remain sheet-like, verdict may be `fix`.

## Strict Style Translation Exception

If the user explicitly asks for a style translation, such as "Pixar style", "3D toy render", "realistic product ad", or "anime remake", the skill may relax `style_fidelity_model.rendering_mode`. It must record that as an `allowed_variation` or a new style variant, not overwrite the original canon style.
