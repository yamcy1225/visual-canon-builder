# v3.7 Reference Preserve Tests

Use these tests when prompt-only style locks are not enough to preserve a character sheet design.

## Goals

1. Add `reference_preserve_model` for exact character-sheet reuse.
2. Add `Reference preserve mode` and `Source cell asset` to `$imagegen` prompt packs when source-cell proportions and style should be preserved.
3. Treat identity-only similarity as failure when silhouette, head/body ratio, limb length, shoe simplification, or line style changes.
4. Escalate from loose `generate` to a source-cell crop plus `identity_preserve` or `edit_target_preserve` after a failed generation.

## SampleMascot Regression

Input: SampleMascot front/full-body sheet cell as source, request angry crossed-arm image with shirt text `MAD`.

Bad output to reject:

- Correct animal species and star pupils.
- Correct `MAD` text.
- Body is taller and more mature than source.
- Legs are elongated.
- Shoes are redesigned with realistic product detail.
- Overall result reads like a new character drawing instead of the same sheet design.

Expected next handoff:

```text
Imagegen execution: mode=edit or generate with identity-preserve reference handling
Reference preserve mode: identity_preserve; source_cell=front full-body from approved SampleMascot sheet; preserve=silhouette, head/body ratio, compact torso, short simplified limbs, simplified shoe shapes, flat 2D line style, detail density; allowed_change=angry expression, crossed arms, shirt text "MAD" only
Source cell asset: required; crop_id=CROP_SampleMascot_FrontFullBody_001; full_sheet_only=blocked_for_exact_preservation
Style fidelity lock: strict match to source cell; flat 2D black-and-white mascot line art, minimal flat grayscale
View/proportion lock: copy the front-cell compact mascot proportions; do not redesign body, legs, shoes, or maturity level
Avoid: identity-only similarity, redesigned taller body, realistic shoe detail, elongated legs, semi-realistic redraw
```

Expected validation:

- If only the full multi-pose sheet was used and no source-cell crop or isolated edit target was provided, handoff should not be `ready` for exact preservation.
- If output looks like the same character reinterpreted by another illustrator, verdict is `reject`.
- If output preserves source-cell proportions and style but the crossed arms are awkward, verdict is `fix`.
- `scripts/build_prompt_pack.py` should return `handoff_status: blocked` when exact preservation requires a source cell but no `source_cell_asset_manifest` is ready.
- `scripts/validate_canon.py` should fail a `ready` handoff that requires source-cell preservation without a ready crop or isolated edit target.

## Source Cell Selection

Use the source cell that best matches the target:

- Full-body pose request: front full-body or closest full-body view.
- Side/back view request: matching side/back cell.
- Expression request: expression head cell plus front full-body for proportions.
- Prop request: prop crop plus character cell if the prop is held or worn.

Create the source-cell manifest with:

```bash
python .codex/skills/visual-canon-builder/scripts/create_source_cell_manifest.py \
  --id CROP_SampleMascot_FrontFullBody_001 \
  --source-image-id Image_SampleMascot_001 \
  --label front_full_body \
  --bbox 10,20,300,600
```
